# Skill: บันทึกข้อความ ECE KMUTNB

สร้างบันทึกข้อความราชการภายในสำหรับภาควิชาวิศวกรรมไฟฟ้าและคอมพิวเตอร์
คณะวิศวกรรมศาสตร์ มหาวิทยาลัยเทคโนโลยีพระจอมเกล้าพระนครเหนือ
ในรูปแบบ .docx พร้อมโลโก้ มจพ. เส้นประตกแต่ง และ TH Sarabun New font
ตามระเบียบงานสารบรรณ พ.ศ. 2565

---

## ขั้นตอนการทำงาน

### 1. สัมภาษณ์ข้อมูล (ถ้าขาด)

ถามข้อมูลต่อไปนี้ที่ยังขาด (ถามรวมครั้งเดียว ไม่ทีละข้อ):

| ข้อมูล | ค่า default |
|--------|-------------|
| **เรื่อง** | (ต้องระบุ) |
| **วัตถุประสงค์ / สาระสำคัญ** | (ต้องระบุ) |
| **งบประมาณ** (ถ้ามี) | – |
| **ตำแหน่งผู้ลงนาม** | อาจารย์ประจำภาควิชาวิศวกรรมไฟฟ้าและคอมพิวเตอร์ |
| **เอกสารแนบ** | ไม่มี |

หากผู้ใช้ให้ข้อมูลเพียงพอแล้ว ให้ข้ามขั้นนี้และสร้างเลย

---

### 2. สร้าง JSON config

แปลงข้อมูลที่ได้รับเป็น JSON config ตามรูปแบบด้านล่าง แล้วบันทึกไว้ที่
`/home/claude/memo_config.json`

```json
{
  "เรื่อง": "...",
  "ที่": "อว ๗๑๕๖.๐๑/",
  "วันที่": "DD",
  "ลงนาม": {
    "ชื่อ": "ผู้ช่วยศาสตราจารย์ ดร.รุสลี่ สุทธวีร์กูล",
    "ตำแหน่ง": "อาจารย์ประจำภาควิชาวิศวกรรมไฟฟ้าและคอมพิวเตอร์"
  },
  "เนื้อหา": [
    {"type": "paragraph", "indent": true, "text": "ด้วย..."},
    ...
  ],
  "เอกสารแนบ": []
}
```

ดูรายละเอียด block types ได้ที่ `references/template_structure.md`

#### ตัวอย่าง config สำเร็จรูปแต่ละประเภท

**ขออนุมัติจัดโครงการ/สัมมนา:**
```json
{
  "เรื่อง": "ขออนุมัติจัดโครงการ...",
  "เนื้อหา": [
    {"type": "paragraph", "indent": true,
     "text": "ด้วย ภาควิชาวิศวกรรมไฟฟ้าและคอมพิวเตอร์ ... มีความประสงค์จัดโครงการ ..."},
    {"type": "list_intro", "indent": true, "text": "โครงการดังกล่าวมีวัตถุประสงค์ ดังนี้"},
    {"type": "numbered_list", "items": ["เพื่อ...", "เพื่อ...", "เพื่อ..."]},
    {"type": "list_intro", "indent": true, "text": "รายละเอียดโครงการโดยสังเขป มีดังนี้"},
    {"type": "detail_list", "items": [
      {"label": "ชื่อโครงการ",      "value": "..."},
      {"label": "กำหนดการ",         "value": "..."},
      {"label": "กลุ่มเป้าหมาย",    "value": "..."},
      {"label": "งบประมาณ",         "value": "... บาท (... บาทถ้วน)"}
    ]},
    {"type": "paragraph", "indent": true,
     "text": "ผลที่คาดว่าจะได้รับ ได้แก่ ..."}
  ]
}
```

**ขออนุมัติจัดซื้อ/จัดจ้าง:**
```json
{
  "เรื่อง": "ขออนุมัติจัดซื้อ...",
  "เนื้อหา": [
    {"type": "paragraph", "indent": true,
     "text": "ด้วย ... มีความประสงค์ขออนุมัติจัดซื้อ ... เพื่อ ..."},
```

**ขออนุมัติเดินทางไปราชการ/เข้าร่วมประชุม:**

### 3. รัน script เพื่อสร้างไฟล์

```bash
SKILL_DIR="<path_to_this_skill>"

python3 "${SKILL_DIR}/scripts/generate_memo.py" \
  --config /home/claude/memo_config.json \
  --template "${SKILL_DIR}/assets/template.docx" \
  --output /home/claude/memo_output.docx
```

> **หา SKILL_DIR**: ดูจาก `available_skills` ที่ระบุ location ของ SKILL.md
> แล้วใช้ parent directory นั้นเป็น `SKILL_DIR`

## ข้อมูลคงที่ของภาควิชา

| ข้อมูล | ค่า |
|--------|-----|
| ส่วนราชการ | คณะวิศวกรรมศาสตร์ ภาควิชาวิศวกรรมไฟฟ้าและคอมพิวเตอร์ |

| ฟอนต์ | TH Sarabun New 16pt |

## การแก้ไขหลังสร้างไฟล์

ส่วนที่ script **ไม่แก้ไขอัตโนมัติ** (ต้องเปิด Word แก้เอง):
- ชื่อผู้ลงนาม (ถ้าต้องการเปลี่ยน)
- เดือนและปีในช่อง "วันที่"
- เลขที่หนังสือในช่อง "ที่"

## Troubleshooting

| ปัญหา | สาเหตุ / วิธีแก้ |
|-------|-----------------|
| `Body anchor paraIds not found` | template.docx ถูกแก้ไขจนเปลี่ยน paraId – ให้ใช้ template ต้นฉบับใน assets/ |
| `pack.py failed` | XML syntax error – ตรวจสอบ text ที่มี `<`, `>`, `&` ให้ escape ก่อน |
| ข้อความขาดหาย | ตรวจสอบ block type ใน config ว่าถูกต้อง |

---

## ข้อมูลเพิ่มเติม

อ่าน `references/template_structure.md` สำหรับ:
- ตำแหน่ง paraId ทุกจุดใน template
- XML pattern ของแต่ละส่วน
- Schema ของ config JSON อย่างละเอียด
